Slide 1 / 14 📘 Review Sesi 9 – Extension Methods
  • Extension methods: Menambahkan fungsi ke tipe yang sudah ada
  • Sintaks: extension Nama on Tipe { ... }
  • Contoh: 'hello'.capitalize()
  • Contoh cepat:
extension IntExt on int {
    bool get isEven => this % 2 == 0;
}

void main() {
    print(10.isEven);  // Output: true
    print(7.isEven);   // Output: false
}
💡 Poin Penting: Extension methods memungkinkan kita menambahkan fungsionalitas baru ke library atau tipe yang tidak bisa kita ubah langsung, tanpa harus membuat subclass.
Slide 2 / 14 🏭 Pengantar Factory Constructors
  • Apa itu? Constructor khusus yang tidak selalu membuat instance baru
  • Keyword: factory
  • Perbedaan dengan generative constructor:
    • Generative: Selalu membuat instance baru
    • Factory: Bisa mengembalikan instance yang sudah ada, atau instance subclass
// Generative constructor (selalu baru)
class Point {
    double x, y;
    Point(this.x, this.y);  // generative → selalu buat instance baru
}

// Factory constructor (bisa return instance existing)
class Logger {
    static final Logger _instance = Logger._internal();
    Logger._internal();
    
    factory Logger() {
        return _instance;  // mengembalikan instance yang sama (singleton)
    }
}
📌 Inti Factory Constructor: Memberikan kontrol penuh atas proses pembuatan objek – apakah membuat baru, mengambil dari cache, atau mengembalikan subclass.
Slide 3 / 14 ✍️ Sintaks Factory Constructor 
class MyClass {
    // Generative constructor (biasa)
    MyClass();
    
    // Factory constructor
    factory MyClass.create() {
        // Bisa return instance yang sudah ada
        // Atau instance dari subclass
        // Atau instance baru
        return MyClass();
    }
}

void main() {
    var obj1 = MyClass();        // Generative
    var obj2 = MyClass.create(); // Factory
}
Slide 4 / 14 🧩 Kasus 1 – Singleton Pattern

Singleton: Hanya satu instance sepanjang aplikasi.

class DatabaseConnection {
    // Private constructor
    DatabaseConnection._internal() {
        print('Creating database connection...');
    }
    
    // Static instance
    static final DatabaseConnection _instance = DatabaseConnection._internal();
    
    // Factory constructor
    factory DatabaseConnection() {
        return _instance;
    }
    
    void connect() {
        print('Connected to database');
    }
}

void main() {
    var db1 = DatabaseConnection();
    var db2 = DatabaseConnection();
    print(identical(db1, db2)); // true - instance yang sama
    db1.connect();
}
Slide 5 / 14 🗃️ Kasus 2 – Caching / ConnectionPool 
class ConnectionPool {
    final String host;
    final int port;
    
    // Cache berdasarkan parameter
    static final Map<String, ConnectionPool> _cache = {};
    
    // Private constructor
    ConnectionPool._(this.host, this.port);
    
    // Factory dengan caching
    factory ConnectionPool(String host, int port) {
        var key = '$host:$port';
        if (_cache.containsKey(key)) {
            print('Returning cached connection for $key');
            return _cache[key];
        }
        print('Creating new connection for $key');
        var connection = ConnectionPool._(host, port);
        _cache[key] = connection;
        return connection;
    }
    
    void query(String sql) {
        print('Executing: "$sql" on $host:$port');
    }
}
💡 Hasil: Memanggil ConnectionPool('localhost', 5432) dua kali akan mengembalikan instance yang sama dari cache.
Slide 6 / 14 🔄 Kasus 3 – Mengembalikan Subclass (Shape) 
abstract class Shape {
    void draw();
    
    // Factory constructor
    factory Shape(String type) {
        switch (type) {
            case 'circle': return Circle();
            case 'square': return Square();
            case 'triangle': return Triangle();
            case 'rectangle': return Rectangle();
            default: throw ArgumentError("Unknown shape: $type");
        }
    }
}

class Circle implements Shape {
    @override void draw() => print("Drawing circle");
}
class Square implements Shape {
    @override void draw() => print("Drawing square");
}
class Triangle implements Shape {
    @override void draw() => print("Drawing triangle");
}
class Rectangle implements Shape {
    @override void draw() => print("Drawing rectangle");
}
Slide 7 / 14 📦 Response dengan Success/Error 
abstract class Response<T> {
    final T? data;
    final String? error;
    
    Response({this.data, this.error});
    
    // Factory untuk success
    factory Response.success(T data) = SuccessResponse<T>;
    
    // Factory untuk error
    factory Response.error(String error) = ErrorResponse<T>;
}

class SuccessResponse<T> extends Response<T> {
    SuccessResponse(T data) : super(data: data);
}

class ErrorResponse<T> extends Response<T> {
    ErrorResponse(String error) : super(error: error);
}

Slide 9 / 14 ⚖️ Factory Constructor vs Static Method 
class Example {
    Example._() {}   // private constructor
    
    // Factory constructor
    factory Example.create() => Example._();
    
    // Static method
    static Example createStatic() => Example._();
}

void main() {
    var e1 = Example.create();        // Factory constructor
    var e2 = Example.createStatic();  // Static method
    
    print(identical(e1, e2)); // false (instance berbeda)
}
Perbedaan: Factory constructor bisa digunakan dalam extends clause dan lebih natural untuk pola desain. Static method lebih fleksibel secara umum tapi tidak bisa digunakan sebagai constructor redirect.
Slide 10 / 14 🎨 Factory untuk Immutable Objects dengan Cache (Color) 
class Color {
    final int red;
    final int green;
    final int blue;
    
    // Cache immutable objects
    static final Map<String, Color> _cache = {};
    
    // Private constructor
    Color._(this.red, this.green, this.blue);
    
    // Factory dengan cache
    factory Color.rgb(int red, int green, int blue) {
        // Validasi
        if (red < 0 || red > 255 || green < 0 || green > 255 || blue < 0 || blue > 255) {
            throw ArgumentError('RGB values must be 0-255');
        }
        var key = '$red,$green,$blue';
        if (!_cache.containsKey(key)) {
            _cache[key] = Color._(red, green, blue);
        }
        return _cache[key];
    }
    
    // Named factories untuk warna umum
    factory Color.red() => Color.rgb(255, 0, 0);
    factory Color.green() => Color.rgb(0, 255, 0);
    factory Color.blue() => Color.rgb(0, 0, 255);
    factory Color.white() => Color.rgb(255, 255, 255);
    factory Color.black() => Color.rgb(0, 0, 0);
    
    @override
    String toString() => 'Color($red, $green, $blue)';
}
💡 Hasil: identical(Color.red(), Color.rgb(255,0,0)) → true (instance sama karena cache)
Slide 11 / 14 📝 Ringkasan & Best Practices
Rangkuman:
  • Factory constructor: factory class.name()
  • Bisa return: instance yang ada, subclass, atau baru
  • Use cases: singleton, caching, object pools, subclass selection
Best Practices:
  • Gunakan factory untuk: Object pooling/caching, Singleton pattern, Complex object creation, Subclass selection
  • Gunakan generative constructor untuk: Simple object creation, Initialization lists, Required parameters
  • Validation: Lakukan validasi di factory sebelum membuat objek
