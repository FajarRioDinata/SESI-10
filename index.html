🏭 Factory Constructors · Konsep & Perbedaan Dasar

✨ SESI 10 — DURASI 200 MENIT

Factory constructor adalah constructor khusus di Dart yang TIDAK selalu membuat instance baru. Ditandai dengan keyword factory. Berbeda dengan generative constructor yang selalu menghasilkan instance baru, factory constructor bisa mengembalikan instance yang sudah ada (cached), instance dari subclass, atau bahkan objek dari pool.

📌 Perbedaan utama:
▪ Generative constructor → selalu membuat objek baru.
▪ Factory constructor → fleksibel: bisa return instance existing, subclass, atau instance baru dari cache. 
class Demo {
    // generative biasa
    Demo();
    
    // factory constructor
    factory Demo.create() {
        return Demo();   // bisa juga return instance tersimpan
    }
}

🧠 Kasus 1 · Singleton Pattern (Satu Instance Global)

Memastikan sebuah class hanya memiliki satu instance sepanjang aplikasi. Kombinasi private constructor + static instance + factory constructor.

class DatabaseConnection {
    // private constructor
    DatabaseConnection._internal() {
        print('Membuat koneksi database...');
    }
    // static instance tunggal
    static final DatabaseConnection _instance = DatabaseConnection._internal();
    
    // factory constructor mengembalikan instance yang sama
    factory DatabaseConnection() {
        return _instance;
    }
    
    void connect() => print('Terhubung ke database');
}

void main() {
    var db1 = DatabaseConnection();
    var db2 = DatabaseConnection();
    print(identical(db1, db2)); // true -> instance sama
}
💡 Pola Singleton sangat berguna untuk resource bersama (database, logger, config).

🗄️ Kasus 2 · Caching & Pooling (ConnectionPool)

Menggunakan cache berbasis parameter untuk mengembalikan instance yang sudah pernah dibuat, menghemat resource.

class ConnectionPool {
    final String host;
    final int port;
    static final Map<String, ConnectionPool> _cache = {};
    
    // private constructor
    ConnectionPool._(this.host, this.port);
    
    // factory dengan cache
    factory ConnectionPool(String host, int port) {
        String key = '$host:$port';
        if (_cache.containsKey(key)) {
            print('✨ Mengembalikan koneksi dari cache untuk $key');
            return _cache[key];
        }
        print('🆕 Membuat koneksi baru untuk $key');
        final conn = ConnectionPool._(host, port);
        _cache[key] = conn;
        return conn;
    }
    
    void query(String sql) => print('Menjalankan: $sql pada $host:$port');
}

🔀 Kasus 3 · Factory yang Mengembalikan Subclass

Factory constructor dapat mengembalikan objek dari tipe turunan (polymorphism) berdasarkan parameter.

abstract class Shape {
    void draw();
    
    factory Shape(String type) {
        switch (type.toLowerCase()) {
            case 'circle': return Circle();
            case 'square': return Square();
            case 'rectangle': return Rectangle();
            default: throw ArgumentError('Bentuk tidak dikenal');
        }
    }
}

class Circle implements Shape {
    @override void draw() => print('Menggambar lingkaran 🔴');
}
class Square implements Shape {
    @override void draw() => print('Menggambar persegi 🟧');
}
class Rectangle implements Shape {
    @override void draw() => print('Menggambar persegi panjang 📏');
}
✅ Keuntungan: Memisahkan logika pembuatan objek kompleks dan menyembunyikan concrete class dari pengguna.

📦 Kasus 4 · Factory untuk Response (Success / Error)

abstract class Response<T> {
    final T? data;
    final String? error;
    
    Response({this.data, this.error});
    
    // Factory success
    factory Response.success(T data) = SuccessResponse<T>;
    
    // Factory error
    factory Response.error(String error) = ErrorResponse<T>;
}

class SuccessResponse<T> extends Response<T> {
    SuccessResponse(T data) : super(data: data);
}

class ErrorResponse<T> extends Response<T> {
    ErrorResponse(String error) : super(error: error);
}

Mudah digunakan: Response.success(userData) atau Response.error('invalid')

💳 Factory untuk Payment Methods (Subclass Selection)

abstract class PaymentMethod {
    void pay(double amount);
    
    factory PaymentMethod(String type) {
        switch (type.toLowerCase()) {
            case 'creditcard': return CreditCardPayment();
            case 'paypal': return PayPalPayment();
            case 'banktransfer': return BankTransferPayment();
            default: throw ArgumentError('Metode tidak valid');
        }
    }
}

class CreditCardPayment implements PaymentMethod {
    @override void pay(double amount) => print('💳 Membayar \$$amount dengan Kartu Kredit');
}
class PayPalPayment implements PaymentMethod {
    @override void pay(double amount) => print('📱 Membayar \$$amount dengan PayPal');
}
class BankTransferPayment implements PaymentMethod {
    @override void pay(double amount) => print('🏦 Transfer bank \$$amount');
}

🎨 Immutable Objects & Cache · Factory Color

Menggunakan factory dengan cache untuk objek immutable (seperti warna) sehingga warna dengan nilai RGB sama akan menggunakan instance yang sama.

class Color {
    final int red, green, blue;
    static final Map<String, Color> _cache = {};
    
    Color._(this.red, this.green, this.blue);
    
    factory Color.rgb(int r, int g, int b) {
        if (r < 0 || r > 255 || g < 0 || g > 255 || b < 0 || b > 255) {
            throw ArgumentError('Nilai RGB 0-255');
        }
        String key = '$r,$g,$b';
        return _cache.putIfAbsent(key, () => Color._(r, g, b));
    }
    
    factory Color.red() => Color.rgb(255, 0, 0);
    factory Color.green() => Color.rgb(0, 255, 0);
    factory Color.blue() => Color.rgb(0, 0, 255);
}

⚖️ Factory Constructor vs Static Method

factory Example()

Bisa mengembalikan subclass, digunakan dalam new expression, lebih natural untuk pola desain. Mendukung polymorphic creation.

static Example.create()

Mirip, tapi tidak bisa digunakan di extends clause atau dalam tear-off. Factory lebih idiomatic untuk constructor alternatif.

class Example {
    Example._();
    factory Example.create() => Example._();    // factory constructor
    static Example createStatic() => Example._(); // static method
}
// Perbedaan subtle: factory bisa digunakan di 'new' (opsional) dan polymorphic creation

📌 Best Practices & Use Cases Factory Constructor

  • Gunakan factory untuk: Object pooling / caching, Singleton, Complex object creation, Subclass selection (polymorphism), Immutable objects dengan cache.
  • Gunakan generative constructor untuk: Pembuatan objek sederhana, initialization lists, required parameters tanpa logika rumit.
  • Validasi terbaik dilakukan di factory sebelum return instance.
🧠 Inti Factory Constructor: Tidak selalu membuat instance baru → mengembalikan instance dari cache, subclass, atau mengontrol alokasi memori secara fleksibel.
🎯 LATIHAN & CHALLENGE (dari materi)

Latihan: 1. ConnectionFactory dengan pooling. 2. NotificationFactory berbasis platform. 3. ShapeFactory dengan cache.
Challenge: AnimalFactory dengan factory method, cache untuk atribut sama, validasi, subclass selection.

🧾 Penjelasan Materi Factory Constructor (Soal & Konsep)

Apa itu Factory Constructor? Constructor yang menggunakan keyword factory dan tidak selalu menciptakan instance baru. Keuntungannya: bisa mengontrol apakah akan membuat objek baru, mengambil dari cache, atau mengembalikan tipe turunan.

Mengapa dibutuhkan? Untuk menghemat memori (caching), implementasi singleton, object pool, dan menyembunyikan logika pembuatan subclass kompleks.

Contoh real: Pada aplikasi payment, PaymentMethod('creditcard') mengembalikan CreditCardPayment, tanpa user harus tahu class konkretnya. Juga pada connection pool: ConnectionPool('localhost',5432) selalu mengembalikan instance yang sama untuk parameter identik.

🔥 Kesimpulan (Fajar Rio Dinata): Factory Constructor adalah teknik powerful dalam Dart untuk desain yang lebih fleksibel, caching, dan enkapsulasi pembuatan objek. Cocok untuk kode yang skalabel dan maintainable.
